ホーム / 車両管理仕様

概要

車両管理モジュールは、配送プラットフォームにおける配送車両の登録・管理・運用を担当します。 各配送企業に属する車両の情報を一元管理し、車両の種類・積載量・稼働状態を追跡します。

本仕様では、車両テーブルの構造、APIエンドポイント、データベーススキーマ、 および実装上の考慮事項について詳細に記載します。

データベーススキーマ

vehicles テーブル

車両情報を管理するテーブルです。各車両は企業に所属し、一意なナンバープレートで識別されます。

カラム名 データ型 制約 説明
id uint PRIMARY KEY 車両ID（自動採番）
company_id uint NOT NULL
FOREIGN KEY		 所属企業ID
plate_number varchar(20) UNIQUE INDEX ナンバープレート（例：品川12あ34-56）
vehicle_type varchar(20) 車両種類（軽トラ、2t、4t、10t、大型）
max_weight float64 最大積載量（kg）
status varchar(20) 状態（active/maintenance/retired）
note text 備考
created_at timestamp 作成日時
updated_at timestamp 更新日時

車両種類（VehicleType）の説明

種類 説明 標準積載量
軽トラ 軽トラック。小規模な配送に使用 350-500kg
2t 2トントラック。中小規模配送に適している 2,000kg
4t 4トントラック。標準的な中型配送車 4,000kg
10t 10トントラック。大型配送に対応 10,000kg
大型 大型トラック（10t以上）。大規模配送に対応 10,000kg以上

ステータス（Status）の説明

ステータス 説明 配送可能
active 稼働中。配送に使用可能な状態 〇 可能
maintenance 整備中。一時的に配送に使用不可 × 不可
retired 廃車。配送に使用不可 × 不可

/api/v1/vehicles

説明： 企業に属する車両一覧を取得します。

実装上の考慮事項

ナンバープレートのUNIQUE INDEXについて

plate_number カラムに UNIQUE INDEX が設定されている理由：

  • 現実世界の一意性： ナンバープレートは日本の法令により一意であり、重複は存在しません。 この特性をデータベースレベルで保証することで、データ整合性を確保します。
  • 配送管理の信頼性： 配送時の車両追跡・識別において、ナンバープレートは重要な識別子です。 一意性を保証することで、誤った車両の登録を防ぎます。
  • クエリ性能： UNIQUE INDEX はクエリのパフォーマンス向上にも貢献します。 ナンバープレートによる検索が頻繁に発生する場合、インデックスは検索を高速化します。
  • アプリケーションロジック削減： UNIQUE INDEX がない場合、アプリケーション側で重複チェックを実装する必要があります。 これはコードの複雑性と保守負荷を増加させます。

企業への属性付け

company_id は NOT NULL 制約を持ちます。 これにより、所有企業を持たない「孤立した」車両の存在を防ぎます。 マルチテナント環境において、データ隔離と権限管理を実現します。

ステータス管理

車両のステータスは論理削除ではなく、ステータスカラムで管理されます。 これにより、廃車となった車両の履歴を保持しながら、 システムから効果的に除外することができます。

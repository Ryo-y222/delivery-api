Delivery Platform 機能仕様書

一覧

プロダクト概要

運送会社同士の助け合いマッチングプラットフォーム。荷主も参加可能。
空車で走るトラックの「帰り荷」問題を解決し、運送業界の空車率を下げる。

2つのモード

モード 対象 概要
ソロモード 運送会社 1社 自社の配車計画・車両・ドライバーを管理。地図で全トラックのルートと予測位置を一覧表示。UIはBqey風のシンプルな左サイドバー。
マルチモード 運送会社 + 荷主 ソロモードで登録した空車区間をマッチング市場に公開。荷主 or 他の運送会社とマッチング。チャット・決済まで完結。

技術スタック

レイヤー技術
バックエンドGo + Gin + GORM + MySQL
フロントエンドReact (TypeScript)
地図Google Maps Platform（Routes API / Maps JavaScript API）
決済Stripe（PaymentIntent + Billing）
インフラDocker / docker-compose

アーキテクチャ（三層構造）

handler/   ← HTTPリクエスト/レスポンスの変換のみ
service/   ← ビジネスロジック
repository/← DB操作のみ（GORMの詳細を閉じ込める）
model/     ← データ構造の定義
dto/       ← レスポンス用の構造体（段階的情報開示）
util/      ← JWT・パスワードハッシュ等のユーティリティ

機能仕様書一覧

1. 認証

API Security

ユーザー登録・ログイン・ログアウト。JWT + HTTP-only Cookie。リフレッシュトークンによるセッション維持。

2. 会社管理

API DB

運送会社の登録・更新。DisplayName（匿名）とName（正式名称）の二段階管理。段階的情報開示の起点。

3. 車両管理

API DB

車番・車種・最大積載量の登録。会社に紐づく。ステータス管理（active / maintenance / retired）。

4. ドライバー管理

API DB

免許情報・資格・事故歴の管理。運行点呼（daily_inspections）は別テーブル。

5. 配車計画

API DB

1台×1日の配車計画を作成。ソロモードの中心機能。

6. 区間管理

API DB

配車計画の各区間（往路・帰路）。cargo_status / visibility でマッチング対象を制御。マッチングの最小単位。

7. 配車ボード & 予測位置

UI API

1つの地図に複数ルートを色分け表示。GPSなしで出発・到着時刻から予測位置を算出。60秒自動更新。

8. マッチング

API DB

荷主→運送会社、運送会社→運送会社のリクエスト。承認/拒否。ステータスマシンで遷移管理。

9. 決済

API Security

Stripe PaymentIntent連携。マッチング承認後、決済を経由しないと完了にならない。Webhook受信。

10. チャット & フィルタリング

API Security

マッチング承認後に自動でチャットルーム作成。電話番号・メール・URLの送信を正規表現でブロック。違反記録。

11. レビュー・評価

API DB

マッチング完了後の相互評価（星1〜5）。会社単位で評価蓄積。

12. サブスクリプション

API Security

Stripe Billing連携。free / standard / premium の3プラン。月間マッチング数制限。

13. 段階的情報開示

API Security

マッチング段階に応じた会社情報の開示制御。DTO切り替えによるレスポンスフィルタリング。

データフロー全体像

[運送会社A] 配車計画作成 → 区間登録（往路: 荷物あり / 帰路: 空車募集中）
                           ↓ visibility = public に設定
[荷主 or 運送会社B] 空車区間を検索 → マッチングリクエスト送信（pending）
                           ↓
[運送会社A] 承認（approved）→ チャットルーム自動作成 → やり取り
                           ↓
[荷主 or B] 決済（payment_pending → succeeded）→ マッチング完了（completed）
                           ↓
              相互レビュー → 会社の rating_avg / total_deals に反映
